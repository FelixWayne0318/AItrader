  ┃                  反转状态机 (Two-Phase Commit)                   ┃
  ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  状态机图解:

  ┌──────────────┐
  │ 初始状态     │
  │ LONG 持仓    │
  │ _pending = ∅ │
  └──────┬───────┘
         │ 收到 SELL 信号 (反转)
         ▼
  ┌──────────────────────────────────────┐
  │ Phase 1: 存储状态                    │
  │ _pending_reversal = {                │
  │   target_side: 'short',              │
  │   target_quantity: qty,              │
  │   old_side: 'long',                  │
  │   submitted_at: now()                │
  │ }                                    │
  │ submit_order(SELL, reduce_only=True) │
  └──────────────┬───────────────────────┘
                 │ on_order_filled
                 │ on_position_closed
                 ▼
  ┌──────────────────────────────────────┐
  │ Phase 2: 检测 _pending_reversal      │
  │ if _pending_reversal:                │
  │   pending = _pending_reversal        │
  │   _pending_reversal = None  # 清空  │
  │   if _get_position() is None:        │
  │     _submit_bracket_order(SHORT)     │
  │   else:                              │
  │     ABORT (残留仓位)                 │
  └──────────────┬───────────────────────┘
                 │ on_order_filled
                 │ on_position_opened
                 ▼
  ┌──────────────┐
  │ 最终状态     │
  │ SHORT 持仓   │
  │ _pending = ∅ │
  └──────────────┘

  边缘情况处理:

  ┌─────────────────────────────────────────────────────────────┐
  │ 情况                        │ 处理                          │
  ├─────────────────────────────────────────────────────────────┤
  │ Phase 2 时仍有仓位          │ ABORT, 发送 CRITICAL 警报     │
  │ Phase 2 提交 Bracket 失败   │ 不开仓, 等待下一信号          │
  │ 平仓订单被拒绝              │ _pending_reversal 保留        │
  │ SL/TP 触发导致平仓          │ 正常进入 Phase 2              │
  │ 手动干预平仓                │ 正常进入 Phase 2              │
  └─────────────────────────────────────────────────────────────┘

  与旧版行为对比:

  旧版 (有竞态条件):
    1. 提交平仓订单
    2. 立即提交开仓订单  ← 问题! 可能在平仓完成前执行
    3. 可能导致双向持仓或订单被拒

  v3.18 (事件驱动):
    1. 存储 _pending_reversal 状态
    2. 提交平仓订单
    3. 等待 on_position_closed 事件
    4. 验证无仓位后开新仓
